    • 비신뢰성

TCP/UDP

  • TCP 특징
    • 전송 제어 프로토콜(Transmission Control Protocol)
    • 연결지향 - TCP 3 way handshake (가상/논리적 연결): 마지막 ACK 보낼때 데이터 함께 전송 가능
    • 데이터 전달 보증
    • 순서 보장: 패킷 순서 틀리면 틀린부분부터 다시 보내라고 요청. 최적화 내부적으로도 가능
    • 신뢰할 수 있는 프로토콜
    • 현재는 대부분 TCP 사용
  • UDP 특징
    • 사용자 데이터그램 프로토콜(User Datagram Protocol)
      • 하얀 도화지에 비유(기능이 거의 없음)
      • 연결지향 - TCP 3 way handshake X
      • 데이터 전달 보증 X순서 보장 X
      • 데이터 전달 및 순서가 보장되지 않지만, 단순하고 빠름 정리
      • IP와 거의 같다. +PORT +체크섬 정도만 추가
        • PORT: 서로 다른 application 구분 가능하도록 함
      • 애플리케이션에서 추가 작업 필요
  • PORT
    • 같은 IP 내에서 프로세스 구분 (IP - 아파트, PORT - 동 호수)
    • 0 ~ 65535 할당 가능0 ~ 1023: 잘 알려진 포트, 사용하지 않는 것이 좋음
    • FTP - 20, 21
    • TELNET - 23
    • HTTP - 80
    • HTTPS - 443
